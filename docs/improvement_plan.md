# 改善対応方針（2026-06 決定）

「株式の分析・記録・想起ツール」としての評価をもとに、機能の過不足を1論点ずつ確認して決定した対応方針。
プロダクトの判断軸は `docs/product_context.md` に従う。

---

## 決定事項一覧

| # | 論点 | 決定 |
|---|------|------|
| 1 | CopoMo（earnings_analysis）廃止の範囲 | **スタンドアロンUIのみ廃止**。EDINET連携は継続 |
| 2 | EDINET開示の想起通知 | **方向性は採用。通知過多・負荷対策の設計を先行** |
| 3 | 売却完結時の振り返り | **控えめに実装**（バッジ+記入ボタン、強制プロンプトなし） |
| 4 | インポート後の理由未記入キュー | **見送り** |
| 5 | タグ別・テーマ別の成績集計 | **trading_dashboard に追加。ダッシュボード自体の内容は再検討** |
| 6 | 全銘柄横断タイムライン | **専用ページとして新設** |
| 7 | バックリンク表示 | **detail の関連タブに統合** |
| 8 | データエクスポート | **CSVのみ実装** |
| 9 | 記入欄の整理 | **DiaryNote を Markdown 化 + memo を廃止し reason に統合** |
| 10 | ナビ・ページ構成 | **メニューを主要5項目に縮退 + stock_list / diary_summary を統合**（investment_hub は現状維持） |

## 各論点の詳細

### 1. CopoMo 廃止の範囲

- 廃止: `/copomo/` 配下のスタンドアロン画面（URL 46本・テンプレート28枚、TDNET 管理/ユーザーUI 含む）とナビの「CopoMo」項目
- 存続: `earnings_analysis` のモデル（`Company`, `DocumentMetadata`, `CompanyFinancialData`, `SentimentAnalysisSession` 等）とサービス層（`EDINETAPIService`, `DisclosureSync`, `XBRLAnalysisService`, `FinancialAnalyzer` 等）
- 理由: detail.html の EDINET タブが上記モデル・サービスを直接 import している（`stockdiary/views.py:3373-3665`）。`DisclosureSync` が `StockDiary.latest_disclosure_date` を更新し、`DiaryNote.source_doc_id` が EDINET 書類を参照しているため、データ層は切り離せない

### 2. EDINET 開示の想起通知（設計検討フェーズ）

push 通知が理想だが、通知過多とサーバー負荷の懸念があるため設計を先行する。検討する負荷対策:

- 1日1回のダイジェスト集約（銘柄ごとの即時 push はしない）
- 重要書類種別のみ対象（有価証券報告書・四半期報告書など、種別でフィルタ）
- ユーザー単位のオプトイン + 通知上限
- django-q の既存スケジューラでバッチ処理し、`DisclosureSync` の差分検知に相乗りする

### 3. 売却完結時の振り返り（控えめ実装）

- 保有数が 0 になった銘柄の detail に「振り返り未記入」バッジと記入ボタンを表示
- 記入時は当時の投資理由（reason）と実現損益を並べて表示
- `DiaryNote.note_type` に `retrospective` を追加
- その場でのモーダル強制表示はしない

### 5. タグ別成績 + ダッシュボード再検討

- 既存の業種別分析（`TradingDashboardView`）と同形式で「タグ別の勝率・実現損益・ROI」を追加。軸（テーマ/マクロ/リスク等）での絞り込み対応
- あわせて trading_dashboard 全体の内容を再検討する。方向性: 証券会社アプリと重複する汎用統計を減らし、「自分の判断の質」（思考の分類×結果）に寄せる

### 6. 全銘柄横断タイムライン（専用ページ）

- DiaryNote・取引・日記作成・振り返りを日付降順に混在表示する専用ページを新設
- 期間・タグ・note_type で絞り込み可能
- 「あの時期の自分は何を考えていたか」を見返す想起の中核画面と位置づける

### 7. バックリンク（関連タブに統合）

- detail.html の「関連銘柄」タブに「この銘柄に触れている記録」セクションを追加
- データソース: `linked_from`（手動リンクの逆方向）+ 他日記本文中の銘柄コード言及（`extract_stock_mentions` を流用）
- 言及箇所のスニペット付きで表示

### 8. CSV エクスポート

- 日記・取引・継続記録をそれぞれ CSV でダウンロードできるユーザー向け機能を追加
- 管理画面限定の `export_csv`（`stockdiary/admin.py`）とは別に、一般ユーザーが使える導線を設ける

### 9. 記入欄の整理

- `DiaryNote.content` の Markdown 化: 表示は既存の `render_markdown` 基盤を流用、入力欄も軽量エディタ化
- `memo` 廃止: フォーム・表示から除去し、既存データは reason 末尾に追記するデータマイグレーションで移行
- 書く場所を「投資仮説（reason）」と「時系列の追記（DiaryNote）」の2層に整理

### 10. ナビ・ページ構成

- 主要ナビを5項目に縮退: 日記一覧 / タイムライン / 関連グラフ / ダッシュボード / 新規作成
- タグ管理・テンプレート・CSV取込・ヘルプ系は「設定・その他」に格納
- `stock_list` と `diary_summary` を統合し、片方の URL はリダイレクト、不要テンプレートは削除
- investment_hub は今回は手を入れない

---

## 見送り・対象外

- 取引インポート後の「理由未記入キュー」（論点4）
- Markdown 一括エクスポート（CSV のみ実装）
- investment_hub の廃止・縮退
- 開示通知の即時実装（設計が固まるまで保留）

## 推奨実施順

1. **低コスト・即効**: バックリンク表示（7）/ CSV エクスポート（8）/ ナビ縮退（10）
2. **記録の質**: memo 統合 + DiaryNote Markdown 化（9）/ 売却振り返りバッジ（3）
3. **新画面・集計**: 横断タイムライン（6）/ stock_list・diary_summary 統合（10）/ タグ別成績 + ダッシュボード再検討（5）
4. **大規模変更**: CopoMo スタンドアロン UI 廃止（1）/ 開示通知の設計→実装（2）
