# カブログ 要件定義書

## 1. システム概要

**プロダクト名**: カブログ  
**種別**: 株式投資日記・振り返りWebアプリケーション（Django / PWA対応）  
**対象ユーザー**: 個人株式投資家（日本株・米国株）

### このアプリが解決する課題

一般的な証券アプリ・家計簿アプリは「売買記録（いつ・何株・いくらで売買したか）」の管理に特化している。しかし**なぜその判断をしたのか**は記録されない。

カブログは「**投資日記**」を中心に据えることで、この課題を解決する。

| | 証券アプリ | カブログ |
|---|---|---|
| 売買記録 | ✓ | ✓ |
| 損益計算 | ✓ | ✓ |
| **買った理由・考えていたこと** | — | **✓** |
| **後からの振り返り・気づき** | — | **✓** |
| **時系列での思考の深掘り** | — | **✓** |

### 価値提供の3層構造

1. **投資日記**（コア）  
   銘柄を買った瞬間の思考・根拠・期待を文章で残す。後から「あのとき何を考えていたか」を正確に振り返れる。

2. **継続記録**  
   保有中の気づき・決算情報・ニュースへの所感を時系列で追記する。思考が深まる過程そのものを記録できる。

3. **取引・分析データ**（補助）  
   損益計算・グラフ・ダッシュボードは「日記の文脈を数字で補強する」ための補助機能として位置づける。

**主目的**: 投資判断の根拠を記録し、振り返りと思考の深化によって投資家としての成長を支援する

### 2.2 株式日記（StockDiary）

#### 基本CRUD

| # | 機能 | 詳細 |
|---|------|------|
| D-1 | 日記作成 | 銘柄コード（任意）・銘柄名・投資理由・業種・タグ・メモ・画像 |
| D-2 | 日記参照 | 詳細画面。取引履歴・継続記録・信用倍率・関連日記タブ |
| D-3 | 日記編集 | 全フィールド更新。画像のクリア機能付き |
| D-4 | 日記削除 | 確認画面あり |
| D-5 | 日記一覧 | ページネーション（4件/ページ）。デフォルトは「保有中」表示 |

#### フィールド仕様

| フィールド | 制約 |
|-----------|------|
| 銘柄名 | 必須、最大100文字 |
| 銘柄コード | 任意、最大50文字 |
| 投資理由（reason） | 任意、最大5000文字、Markdown対応 |
| メモ（memo） | 任意、最大1000文字 |
| 業種 | 任意、最大50文字 |
| 画像 | JPEG/PNG/GIF/WebP、最大10MB、保存時に最大800×600・WebP圧縮 |

#### 自動集計フィールド（取引保存時に再計算）

- 現在保有数 / 平均取得単価 / 総取得原価
- 実現損益（FIFO方式）
- 累計購入数・累計売却数・累計購入額・累計売却額
- 取引回数 / 初回購入日 / 最終取引日
- 信用取引除外の現物のみ集計（同名フィールドを `cash_only_` プレフィックスで別管理）

#### 保有ステータス区分

| ステータス | 条件 |
|-----------|------|
| 保有中（holding） | `current_quantity > 0` |
| 売却済み（sold） | `transaction_count > 0` かつ `current_quantity == 0` |
| メモ（memo） | `transaction_count == 0` |
| 信用売り（short） | `current_quantity < 0` |

### 2.3 取引管理（Transaction）

| # | 機能 | 詳細 |
|---|------|------|
| T-1 | 取引追加 | 種別（購入/売却）・日付・単価・数量・メモ・信用取引フラグ |
| T-2 | 取引編集 | 全フィールド更新、保存後に日記の集計再計算 |
| T-3 | 取引削除 | 削除後に集計再計算 |
| T-4 | 売却バリデーション | 保有数超過の売却をエラー |

#### 損益計算ルール

- **通常売却**: 平均取得単価法（FIFO的実装）で実現損益を計算
- **信用売り**: `current_quantity` がマイナスになることを許容
- **返済買い**: マイナス保有に対する買いで損益確定
- 取引順序は `transaction_date` → `created_at` の昇順

#### バリデーション

| フィールド | 制約 |
|-----------|------|
| 単価 | 正の数、最大9,999,999.99円 |
| 数量 | 正の整数、最大99,999,999株 |
| メモ | 最大500文字 |

### 2.6 フィルタリング・ソート・検索

#### フィルター

| 種別 | 選択肢 |
|------|--------|
| ステータス | 保有中 / 売却済み / メモ / すべて（デフォルト: 保有中） |
| タグ | ユーザータグ単一選択 |

| ハッシュタグ | `@タグ名` 形式の本文内タグ |
| 購入日範囲 | 1週/1ヶ月/3ヶ月/6ヶ月/1年 |
| 取引日範囲 | 同上（Transaction.created_at基準） |

#### ソート

#### 検索（全文）

- 銘柄名・コード・投資理由・メモ・業種のキーワード部分一致
- `ExploreView`（Obsidian風全文検索画面）：日記・継続記録・ハッシュタグ横断検索

- **現物取引のみ**を集計対象とする

## 3. 非機能要件

### 3.1 認証・認可

- 全エンドポイントに `@login_required` または `LoginRequiredMixin` を適用
- オブジェクトレベルの認可：`user=request.user` でクエリを必ずフィルタリング
- CSRF対策：Django標準CSRFミドルウェア、AJAX/HTMXリクエストにも適用

### 3.2 パフォーマンス

- `select_related` / `prefetch_related` で N+1 クエリを防止
- `defer` で不要フィールドを除外（一覧取得時）
- 銘柄コード→日記IDマップをDjango LocMemCacheで5分キャッシュ（mention_map）
- DBインデックス：`user+first_purchase_date`, `user+stock_symbol`, `user+current_quantity`, `user+updated_at`

### 3.3 画像処理

- Pillow でリサイズ・WebP変換（日記: 800×600、継続記録: 600×400）
- ストレージ: `django.core.files.storage.default_storage`
- 配信: 認証付きビュー（`ServeImageView`）経由、`Cache-Control: private, max-age=3600`
- サムネイル: URLパラメータ `?thumbnail=1&w=&h=` でオンザフライ生成

### 3.4 UI/UX

- レスポンシブ対応（Bootstrap 5.3）
- HTMXによる部分更新（日記リスト・タブコンテンツ）
- Markdown レンダリング（投資理由フィールド）
- PWA対応（Service Worker・Webプッシュ）

### 3.5 エラーハンドリング

- Django ValidationError を forms / views 両レイヤーでキャッチしメッセージ表示
- API・Ajaxレスポンスは `{"success": bool, "message/error": str}` に統一
- `ObjectNotFoundRedirectMixin` で404時にホームへリダイレクト
- CSRF失敗時のカスタムハンドラー（テストアカウント向け説明付き）

## 4. データモデル関連図（概要）

```
CustomUser
  ├── StockDiary (1:N)
  │     ├── Transaction (1:N)
  │     ├── StockSplit (1:N)
  │     ├── DiaryNote (1:N)
  │     ├── DiaryNotification (1:N)
  │     ├── linked_diaries (M:M self)
  │     └── tags (M:M Tag)
  ├── Tag (1:N)
  ├── AnalysisTemplate (1:N)
  │     ├── TemplateCompany (through: CompanyMaster)
  │     └── TemplateMetrics (1:N)
  ├── PushSubscription (1:N)
  └── NotificationLog (1:N)

CompanyMaster (独立マスタ)
MetricDefinition (独立マスタ)
IndustryBenchmark (MetricDefinition FK)
CompanyScore (AnalysisTemplate + CompanyMaster)
```

## 7. 主要制約・前提条件

- Python / Django フレームワーク
- DB: Django ORM（RDBMS前提）
- `margin_trading` アプリはオプション（未インストール時はフォールバック）
- 米国株対応（Yahoo Finance経由）、為替換算なし
- ランディングページ（`/`）は未ログインでも閲覧可能
