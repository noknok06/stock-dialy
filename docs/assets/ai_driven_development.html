カブログ AI駆動開発 ケイパビリティマップ

コンテキスト × 構造 × 検証 ── AI が「忘れず・取り違えず・自分で確かめて」開発できる基盤（本セッション構築分）
別システムの「コンテキストレイヤー」を、AI 開発者へのコンテキスト供給として翻案したもの
到達点（North Star）
① AI が規約を忘れても壊れない・ ② 指標・概念の定義を取り違えない・ ③ 自分の変更を自分で検証できる
1
コンテキスト供給AI に与える「正本」＝オントロジー＋セマンティック＋規約
オントロジー 概念の意味定義
  • reason＝背景（会社／ニュース／テーマ・遅く変わる）
  • topic＝まとめスレッド全般（購入理由・決算・ニュース）
  • note_type＝種類・絞り込み軸／@tag＝横断テーマ
  • Thesis → Verdict → Learning（検証ループ）
  • 正本: diary_recording_redesign.md / product_context.md
セマンティックレイヤー 指標定義の正本
  • services.metrics：的中／勝ち／2×2象限／閾値
  • 純粋関数・定数（モデル非依存）
  • Verdict・karte_service が参照（重複排除）
規約（ルールエンジン） CLAUDE.md
  • View／HTMX／URL パターン・ディレクトリ規約
  • サービス層の表（metrics を正本として登録）
  • 禁止事項・集計の不変条件を明文化
2
構造的不変条件規約ではなく構造で守る＝忘れても壊れない
集計の不変条件 AggregateService
  • Transaction.save()/delete() が自動再集計（単体）
  • AggregateService.deferred()＝一括操作の唯一の入口
  • ブロック出口で必ず1回だけ再集計（O(N)→O(1)）
定義の単一正本 DRY
  • 象限ラベル・閾値の重複を排除（metrics に集約）
  • reason＝背景 を全レイヤで一貫
  • model／form／template／TagBook まで統一
変更の分離 安全な刻み
  • 自分の変更だけを migration 化
  • 既存ドリフト（memo 等）と混ぜない
  • 大規模リプレイスをしない
3
検証ループAI が自分の変更を確かめ、自己修正する
自動テスト pytest
  • 全 546 tests green / config.test_settings
  • --nomigrations / --reuse-db で高速ループ
回帰でオントロジーを固定 定義を退行させない
  • deferred の保証 3本
  • metrics 純粋関数・タクソノミ 10ケース
  • 「背景」ラベルの退行防止 2本
実機・CI fast feedback
  • run-app / verify スキルで画面確認
  • GitHub Actions：push／PR で pytest 自動実行
作業単位・運用レイヤ（AI にタスクを渡す器）
論点管理：improvement_plan.md が「論点ごとに決定・状態」を保持。AI へ渡す自然なタスク単位
設計ファースト：コードより先にオントロジーを doc で固める（改訂ノートで来歴を残す）
意図を残す：小さなコミット粒度＋「なぜ」を docstring／コミットメッセージに記述
設計原則
構造で守る 定義は1箇所（正本） 既存ファイル優先 大規模リプレイスしない 変更にはテスト コンテキスト駆動 忘れても壊れない
